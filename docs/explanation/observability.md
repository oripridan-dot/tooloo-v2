# Observability in TooLoo V2

TooLoo V2 emphasizes robust observability through structured logging and distributed tracing, enabling clear insights into pipeline execution and debugging.

## Structured Logging

All logs generated by TooLoo V2 components are structured in JSON format. This facilitates easier parsing, filtering, and analysis by logging aggregation tools.

*   **Correlation IDs**: Each log entry includes correlation IDs (e.g., `trace_id`, `span_id`, `request_id`) to trace a request or workflow across different services or components.
*   **Log Levels**: Standard logging levels (`DEBUG`, `INFO`, `WARNING`, `ERROR`, `CRITICAL`) are supported and configurable via `engine/config.py`.

## Distributed Tracing (OpenTelemetry)

TooLoo V2 integrates OpenTelemetry, the de-facto standard for distributed tracing. This allows for:

*   **End-to-End Visibility**: Track requests as they flow through the various stages of the DAG pipeline.
*   **Performance Analysis**: Identify bottlenecks and areas for optimization by analyzing trace durations.
*   **Dependency Mapping**: Understand the relationships and interactions between different tasks.

Traces are generated for task executions, engine operations, and inter-component communication.

## Visualizing Data

*   **Mermaid.js**: Architectural diagrams and DAG flows are often visualized using Mermaid.js syntax, embedded directly in documentation.
*   **Screenshots**: The `render_screenshot` tool can be used to capture visual states of HTML components or UI elements during pipeline execution for manual inspection or A/B testing.

By combining structured logging and distributed tracing, TooLoo V2 provides deep visibility into its autonomous operations.
